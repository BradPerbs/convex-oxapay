# Contributing to `convex-oxapay`

Thanks for considering a contribution! This package follows the same patterns as the official Convex components ([`@convex-dev/stripe`](https://github.com/get-convex/stripe), [`@convex-dev/polar`](https://github.com/get-convex/polar)).

## Development setup

```bash
git clone https://github.com/bradperbs/convex-oxapay
cd convex-oxapay
npm install
npm run dev:backend      # convex dev with component typecheck
npm run dev:frontend     # vite dev server for example/
npm test                 # vitest
npm run typecheck        # tsc --noEmit
npm run lint
```

## Project layout

```
src/
  client/                   # The consumer-facing API
    index.ts                # OxaPay class + registerRoutes
    http.ts                 # OxaPayClient (raw fetch wrapper)
    webhook.ts              # HMAC-SHA512 verifier
    helpers.ts              # Pure crypto + URL helpers
    types.ts                # OxaPay envelope + payload types
    errors.ts               # OxaPayApiError, WebhookVerificationError
  component/                # The Convex component
    convex.config.ts        # defineComponent("oxapay")
    schema.ts               # customers, payments, payouts, processedWebhooks
    lib.ts                  # All queries/mutations/actions
    util.ts                 # Translation between OxaPay payloads and DB rows
    _generated/             # Generated by `npx convex codegen`
  react/                    # Optional React helpers
  test.ts                   # convex-test registration helper
example/                    # Live demo app
```

## Regenerating `_generated/` files

The `src/component/_generated/*.ts` files are **committed to git**. They are hand-maintained to mirror what `npx convex codegen --component-dir ./src/component` would produce. Editing them is fine, but if you've made non-trivial changes to `lib.ts` or `schema.ts`, you should refresh them by running:

```bash
npm run build:codegen
```

This requires a configured Convex deployment (`npx convex dev` to set up `CONVEX_DEPLOYMENT` in `.env.local`). It will overwrite the committed files in place. Review the diff before committing.

**Do NOT** call `build:codegen` from `prepublishOnly` or CI. Those paths must work without a Convex login. They run plain `npm run build` against the committed files.

## Tests

Tests use `vitest` with the `edge-runtime` environment. Component-side tests use `convex-test`.

```bash
npm test               # one-shot
npm run test:watch     # watch mode
npm run test:coverage  # with coverage report
```

When adding a new endpoint, add tests in all relevant layers:

- **`src/client/http.test.ts`**: mock the HTTP request, assert headers/body/URL shape.
- **`src/client/webhook.test.ts`**: only when changing webhook semantics.
- **`src/component/util.test.ts`**: for any new conversion helper.
- **`src/component/lib.test.ts`**: for any new mutation/query.

## Code style

- TypeScript strict mode on. No `any` in the public API; internal `any` requires a justifying comment.
- Two-space indent, double quotes, trailing commas (enforced by Prettier).
- Imports are ordered: stdlib → third-party → relative `./`. Type-only imports use `import type`.

## Publishing

```bash
npm version patch
npm publish
git push --follow-tags
```

`prepublishOnly` runs `build:clean` which executes codegen, build, tests, lint, and typecheck.
